# Evals 设计（子文档，占位）

本文档将细化评估与回归：用例格式、判分规则（触发/动作序列/输出）、以及报告产物（JSON/Markdown）。

## 1. 用例格式

- input：用户请求
- expected：期望技能选择与关键动作约束
- constraints：预算与禁止工具等安全约束

## 2. 指标

- 触发：precision/recall（可用例内标注正确技能集合）
- 执行：动作序列合规、预算消耗、错误分布
- 输出：结构与关键要点满足度

## 3. MockModel 在评估中的定位

- 用于离线验证 Agent Core/权限/分发/加载器的正确性
